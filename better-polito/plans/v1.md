Part 5: AI Features вҖ” Deep Analysis
рҹҸҶ Tier 1: High Value, Low Effort (Build First)



рҹӨ– AI Chatbot over Guides & Services (RAG)
What it does: Natural language Q&A вҖ” "Where do I renew my student ID?" вҶ’ instant answer from guides data


How it works:


Data source: The app already fetches useGetGuides() which returns structured guide content (sections, fields, HTML text)
Indexing pipeline (one-time + periodic refresh):
Fetch all guides via CoursesApi.getGuides()
Parse HTML content вҶ’ plain text chunks (500-token windows with overlap)
Generate embeddings via OpenAI text-embedding-3-small
Store in Pinecone/pgvector with metadata (guide title, section, URL)
Query flow:
User types question вҶ’ embed question вҶ’ vector similarity search (top 5 chunks)
Feed chunks as context to GPT-4o-mini / Claude Haiku
Stream response with source citations
Data available: Guides, news, FAQs from ticket system (useSearchTicketFaqs), services descriptions
Effort: ~3 days | Value: Very high вҖ” most common student pain point



рҹ“… AI Study Planner
What it does: Analyze exam dates + course workload вҶ’ suggest optimal study blocks


How it works:


Data sources (all already available via API):
useGetExams() вҶ’ upcoming exam dates and types
useGetCourses() вҶ’ enrolled courses with credit weights
useGetGrades() вҶ’ historical performance (strength/weakness per subject)
useGetDeadlineWeek() вҶ’ assignment deadlines
Booking data вҶ’ existing commitments
AI logic:
Aggregate all time-bound data into a structured prompt
LLM generates a week-by-week study plan with:
Study blocks allocated proportionally to exam difficulty Г— credits Г— days remaining
Spaced repetition scheduling
Breaks and buffer time
Output as a structured JSON вҶ’ render as an interactive calendar
User interaction: Adjustable parameters (study hours/day, priority courses, existing commitments)
Effort: ~4 days | Value: Very high вҖ” directly impacts academic success



рҹ“ҡ Course Material Summarizer
What it does: Upload/select a course PDF вҶ’ get AI summary, key concepts, study notes


How it works:


Data source: The app already has useGetCourseFiles(courseId) which lists all course files (PDFs, slides, docs)
Pipeline:
User selects a file from the file browser
Backend downloads file via API client вҶ’ extract text (pdf-parse / pdfjs-dist)
Send to LLM with structured prompt: "Summarize this lecture into key concepts, definitions, and potential exam questions"
Stream markdown response
Advanced: Store summaries for reuse, allow "ask questions about this document"
Effort: ~3 days | Value: High вҖ” saves hours of reading


рҹҘҲ Tier 2: Medium Effort, High Value (Build Second)
4. рҹ“Ҡ Grade Predictor & Analytics Dashboard
What it does: Visualize GPA trajectory, predict outcomes, flag at-risk courses


Data available:


useGetGrades() вҶ’ full transcript with dates, scores, credits, course names
useGetProvisionalGrades() вҶ’ pending results
useGetCourseStatistics() вҶ’ grade distributions per course (already aggregated!)
Student enrollment year вҶ’ time-to-degree calculation
AI features:


GPA Trajectory: Plot historical GPA over semesters with trendline + projected graduation GPA
Grade Prediction: For upcoming exams, use: historical grades in similar courses + course statistics (mean, std dev from getCourseStatistics) + student's average вҶ’ Bayesian estimate of likely outcome
Burnout Detection: Analyze schedule density (lectures + exams + deadlines per week) вҶ’ flag overloaded weeks with >X hours of commitments
Time-to-Degree: Based on current credit accumulation rate, project graduation date
Effort: ~5 days | Value: High вҖ” proactive academic guidance



рҹҺ« AI Ticket Triage & FAQ Auto-Answer
What it does: Before submitting a support ticket, suggest existing solutions


How it works:


Existing data: useSearchTicketFaqs(search) already searches FAQs, useGetTicketTopics() has categories
Enhancement:
User starts typing a ticket description
Debounced semantic search over FAQ embeddings (same RAG infra as chatbot)
Surface top 3 matching FAQs inline: "Did you mean...?"
Auto-categorize ticket via LLM classification (from the existing topic taxonomy)
Suggest solutions from guides before submission
Effort: ~3 days (reuses chatbot RAG infrastructure) | Value: Reduces ticket volume



рҹ“қ AI Flashcard Generator

Output as interactive flip cards with spaced repetition (Leitner system)
Store in localStorage, sync across sessions
Effort: ~3 days | Value: Medium-high вҖ” proven study technique
