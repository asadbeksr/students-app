Part 5: AI Features — Deep Analysis
🏆 Tier 1: High Value, Low Effort (Build First)



🤖 AI Chatbot over Guides & Services (RAG)
What it does: Natural language Q&A — "Where do I renew my student ID?" → instant answer from guides data


How it works:


Data source: The app already fetches useGetGuides() which returns structured guide content (sections, fields, HTML text)
Indexing pipeline (one-time + periodic refresh):
Fetch all guides via CoursesApi.getGuides()
Parse HTML content → plain text chunks (500-token windows with overlap)
Generate embeddings via OpenAI text-embedding-3-small
Store in Pinecone/pgvector with metadata (guide title, section, URL)
Query flow:
User types question → embed question → vector similarity search (top 5 chunks)
Feed chunks as context to GPT-4o-mini / Claude Haiku
Stream response with source citations
Data available: Guides, news, FAQs from ticket system (useSearchTicketFaqs), services descriptions
Effort: ~3 days | Value: Very high — most common student pain point



📅 AI Study Planner
What it does: Analyze exam dates + course workload → suggest optimal study blocks


How it works:


Data sources (all already available via API):
useGetExams() → upcoming exam dates and types
useGetCourses() → enrolled courses with credit weights
useGetGrades() → historical performance (strength/weakness per subject)
useGetDeadlineWeek() → assignment deadlines
Booking data → existing commitments
AI logic:
Aggregate all time-bound data into a structured prompt
LLM generates a week-by-week study plan with:
Study blocks allocated proportionally to exam difficulty × credits × days remaining
Spaced repetition scheduling
Breaks and buffer time
Output as a structured JSON → render as an interactive calendar
User interaction: Adjustable parameters (study hours/day, priority courses, existing commitments)
Effort: ~4 days | Value: Very high — directly impacts academic success



📚 Course Material Summarizer
What it does: Upload/select a course PDF → get AI summary, key concepts, study notes


How it works:


Data source: The app already has useGetCourseFiles(courseId) which lists all course files (PDFs, slides, docs)
Pipeline:
User selects a file from the file browser
Backend downloads file via API client → extract text (pdf-parse / pdfjs-dist)
Send to LLM with structured prompt: "Summarize this lecture into key concepts, definitions, and potential exam questions"
Stream markdown response
Advanced: Store summaries for reuse, allow "ask questions about this document"
Effort: ~3 days | Value: High — saves hours of reading


🥈 Tier 2: Medium Effort, High Value (Build Second)
4. 📊 Grade Predictor & Analytics Dashboard
What it does: Visualize GPA trajectory, predict outcomes, flag at-risk courses


Data available:


useGetGrades() → full transcript with dates, scores, credits, course names
useGetProvisionalGrades() → pending results
useGetCourseStatistics() → grade distributions per course (already aggregated!)
Student enrollment year → time-to-degree calculation
AI features:


GPA Trajectory: Plot historical GPA over semesters with trendline + projected graduation GPA
Grade Prediction: For upcoming exams, use: historical grades in similar courses + course statistics (mean, std dev from getCourseStatistics) + student's average → Bayesian estimate of likely outcome
Burnout Detection: Analyze schedule density (lectures + exams + deadlines per week) → flag overloaded weeks with >X hours of commitments
Time-to-Degree: Based on current credit accumulation rate, project graduation date
Effort: ~5 days | Value: High — proactive academic guidance



🎫 AI Ticket Triage & FAQ Auto-Answer
What it does: Before submitting a support ticket, suggest existing solutions


How it works:


Existing data: useSearchTicketFaqs(search) already searches FAQs, useGetTicketTopics() has categories
Enhancement:
User starts typing a ticket description
Debounced semantic search over FAQ embeddings (same RAG infra as chatbot)
Surface top 3 matching FAQs inline: "Did you mean...?"
Auto-categorize ticket via LLM classification (from the existing topic taxonomy)
Suggest solutions from guides before submission
Effort: ~3 days (reuses chatbot RAG infrastructure) | Value: Reduces ticket volume



📝 AI Flashcard Generator
What it does: Auto-create study flashcards from lecture materials


How it works:


Reuse the document parsing from Course Summarizer
LLM prompt: "From this lecture content, generate 10-20 flashcards as Q&A pairs. Focus on definitions, key concepts, formulas, and likely exam questions."
Output as interactive flip cards with spaced repetition (Leitner system)
Store in localStorage, sync across sessions
Effort: ~3 days | Value: Medium-high — proven study technique
